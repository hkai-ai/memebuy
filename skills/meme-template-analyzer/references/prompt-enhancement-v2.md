# promptEnhancement v2 运行规范

本文件约束模板的后端二次改写层。它适用于槽位编辑和全文自由编辑，每次生成都执行。

## 运行顺序

1. 用 `inputSchema` 渲染 `promptTemplate`，得到完整基础提示词。
2. 用户选择全文编辑时，先把用户改写合并进基础提示词；用户指定的开放内容拥有最高内容语义权限。
3. 执行 `promptEnhancement`，只恢复参考图负责的呈现维度和模板语义锚点。
4. 将最终提示词、`referenceImage` 和用户图片一起交给图像模型。

全文编辑后进入增强层的基线必须已经体现用户改写。禁止从旧槽位值重建基线，也禁止把用户要求替换的主体、道具或配饰降级为装饰。

## 权限边界

- 开放内容由用户决定，也就是“是什么”：主体身份、道具、配饰、随身物、嵌套内容和文字内容。
- 参考图决定“怎么呈现”：构图、画幅、裁切、镜头景别、机位、姿态、背景环境、留白、画风、媒介、材质、整体色调和光影。
- 固定 IP、真人肖像或品牌角色需要锁定身份时，不创建主体槽，并设置 `identity_authority: high`。

## instruction

控制在约 300 字内，只说明：输出完整可执行提示词；参考图的呈现权限；开放内容的用户权限；只输出干净成图。参考图权限必须显式排除开放内容和主体身份，不要用文字复述参考图的具体画面。

禁止：

- 复述参考图中的具体主体、道具、身体特征、位置取值或场景细节。
- 要求最终提示词复述“以参考图为基准”等元指令。
- 使用无限定的“参考图最高权限”压制开放内容。

## lockedConstraints

每条只指名一个视觉维度，建议不超过 40 字。常用三条：

```json
[
  "沿用参考图的画幅、裁切、留白、镜头景别与元素位置比例",
  "沿用参考图的媒介质感、材质表现与色彩关系",
  "沿用参考图的前景背景层级、遮挡关系与阅读顺序"
]
```

删除“图像依据：”“具体为”“即……”等复述从句。开放槽位的默认主体、道具或配饰不得出现在此数组中。

## preserve

只保存换掉主体后仍须成立的语义锚点，例如“盒内主体必须被盒沿遮挡”“主体必须成为食物结构的一部分”。没有语义锚点时使用空数组。不得复制 `lockedConstraints`。

## inputSchema

- 同款模式只开放内容槽：主体、道具、配饰、文字和嵌套内容。
- 不开放整体风格、背景环境、姿态、机位、景别、画幅、构图、整体色调、光影、媒介和材质。
- 对象自身的内容属性可以开放，例如“领结颜色”；整体色调和背景颜色保持锁定。
- `label` 使用明确中文内容名词。运行时以 `prompt` 和 `subject` 的 label 构造 `openSlotLabels`。
- 每个开放槽位必须自然出现在 `promptTemplate`。不得靠转换器追加“标签：placeholder”清单。
- 当前运行时未把纯 `select` label 加入 `openSlotLabels`；同款模板使用 `prompt.suggestions` 或 `subject.text.suggestions`。

## 当前运行时事实

Schema 仍要求填写 `stageKey`、`referenceField` 和 `output`，但当前代码不消费这些字段：运行时固定走 `gallery.template_rewrite`，读取 `parsed.prompt`，增强层本身是纯文本 stage。保留字段用于契约兼容，不依赖它们产生行为。

`metadata.templateSource.authority` 当前也是声明信息。真实裁决依赖带作用域的 instruction、开放槽位 label 和运行时合并逻辑。

## 校验

- instruction 同时声明参考图呈现权限和用户开放内容权限。
- locked constraints 只写维度、每条不超过 40 字、无参考图内容复述。
- preserve 不与 locked constraints 重复。
- 开放槽位不包含呈现维度，也不被 locked constraints 锁定。
- 所有开放槽位均被 `promptTemplate` 使用。
- 全文编辑回归中，“增强前基线”应体现用户改写，最终提示词不得恢复旧主体、旧道具或旧配饰。
