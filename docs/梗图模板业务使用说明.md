# 梗图模板业务使用说明

这份说明给业务、运营、内容和设计同学使用。目标是让你们可以直接通过 agent 完成梗图模板分析、用户上传图替换、创意等级对比和生成质量测试。

## 适用场景

适合使用这套流程的情况：

- 你看到一张梗图，想把它做成可复用模板。
- 你希望用户上传宠物、商品、人物或物件照片后，套进某个梗图模板。
- 你想比较同一个模板在不同创意等级下的变化。
- 你想确认生成结果到底有没有使用用户上传图、有没有使用原梗图作为参考。
- 你想把一个梗图模板交给后续生成系统或设计同学继续生产。

不适合直接使用的情况：

- 只想随便生成一张趣图，不关心模板是否稳定。
- 不需要保留原梗图的笑点、构图、风格或荒诞感。
- 用户上传图完全看不出主体是什么。

## 使用前准备

发给 agent 前，尽量准备这些材料：

1. 原始梗图
   - 可以是上传图片、截图、本地图片路径或图片链接。
   - 用来判断这个梗图的笑点、风格、构图和可替换部分。

2. 用户上传图
   - 例如宠物、商品、人物或物件照片。
   - 图片低清也可以，只要主体还能识别。
   - 如果用户图很糊、主体太小、遮挡严重，最好补充 1-2 张参考图。

3. 目标任务
   - 是只分析模板，还是要生成替换方案。
   - 是要高保真，还是要更自由的创意版本。
   - 是否要输出 `lv1-lv5` 对比。

4. 业务边界
   - 是否允许换场景。
   - 是否允许换动作。
   - 是否允许加文字。
   - 是否必须保持原图风格。
   - 是否需要避免品牌、版权角色、敏感场景或不合适表达。

## 推荐工作流

### 第一步：先分析原梗图

先让 agent 看懂原梗图，不要一上来就要求生成。

可以这样说：

```text
使用 $meme-template-analyzer 分析这张梗图。
请用业务语言说明：
1. 这个梗图的核心笑点是什么；
2. 哪些元素必须保留；
3. 哪些元素可以替换；
4. 如果用户上传宠物或商品图，适合替换哪个位置；
5. 不要生成图片，先给我结论。
```

你要重点看：

- agent 是否说清楚这个梗为什么好笑。
- agent 是否区分了“可替换内容”和“必须保留的模板特征”。
- agent 是否没有乱加外部故事，比如办公室、打工人、系统报错等。

### 第二步：确定创意等级

当你想控制变化程度时，让 agent 输出 `lv1-lv5`。

推荐理解方式：

| 等级 | 业务含义 |
| --- | --- |
| `lv1` | 只做最小替换，基本保留原模板。 |
| `lv2` | 小范围变化，例如道具、颜色、配饰。 |
| `lv3` | 可以变化动作、表情、姿态，但仍像同一张梗图。 |
| `lv4` | 可以换同语义场景，例如休闲场景换成沙滩、泳池、露台。 |
| `lv5` | 可替换维度最多，但仍必须保留原梗图的风格、构图关系和笑点公式。 |

可以这样说：

```text
基于刚才的梗图模板，给我 lv1-lv5 的业务版本说明。
要求：
1. 每个等级说明会变化什么；
2. 每个等级说明不能变化什么；
3. 不要加入模板里没有的外部剧情；
4. 不要生成图片，先输出给我确认。
```

### 第三步：加入用户上传图

当用户上传具体宠物、商品、人物或物件图时，一定要让 agent 明确说明是否使用这张图作为主体参考。

可以这样说：

```text
现在把用户上传的这张图作为主体参考，套入刚才的梗图模板。
请先不要生成图片，先告诉我：
1. 是否会使用用户上传图作为主体参考；
2. 原梗图是否会作为生成参考图；
3. 如果不直接使用原梗图，会保留哪些模板特征；
4. 用户上传图如果低清，哪些身份特征还能保留；
5. 本次生成更适合判断创意方向，还是适合判断身份一致性。
```

你要重点看：

- agent 是否明确说“使用用户上传图作为主体参考”。
- agent 是否明确说“原梗图是直接作为参考图，还是只提取风格和构图”。
- 如果用户图低清，agent 是否降低身份一致性的承诺，而不是假装能完全一致。

### 第四步：生成前要求参考图说明

每次正式生成前，都要求 agent 给出参考图使用说明。

推荐要求：

```text
正式生成前，请先给我一段业务可读的参考图使用说明。
必须说明：
1. 本次是否使用用户上传图；
2. 本次是否使用原梗图；
3. 如果使用原梗图，它的作用是什么；
4. 如果不使用原梗图，会如何保留原模板；
5. 当前结果主要验证什么。
```

业务人员验收时，必须能回答：

- 这张图是不是用了用户上传图。
- 这张图是不是用了原梗图。
- 这张图到底在测试身份一致性、模板稳定性，还是只是创意方向。

如果回答不上来，不要把结果当成最终质量判断。

## 三种测试方式

当你要判断质量是否稳定时，让 agent 按三种方式测试。

### 方式一：只用文字

不使用用户上传图，也不使用原梗图。

用途：

- 看没有参考图时会偏到什么程度。
- 作为最低基准。

业务判断：

- 如果只用文字也能稳定，说明模板很好描述。
- 如果漂移很大，说明后续必须依赖参考图。

### 方式二：只用用户上传图

使用用户上传图作为主体参考，不直接使用原梗图。

用途：

- 判断用户主体是否能保留。
- 判断真实产品链路是否可行。
- 避免原梗图把结果拉回源主体。

这是最推荐的默认测试方式。

### 方式三：用户上传图 + 原梗图

同时使用用户上传图和原梗图。

用途：

- 判断原梗图是否能增强构图和风格。
- 观察是否出现源主体泄漏。
- 观察是否复制原图文字、水印、Logo 或其他无关痕迹。

业务判断：

- 如果构图明显更稳，且没有源图泄漏，可以考虑使用。
- 如果出现原图主体、原图文字或奇怪痕迹，应回到“只用用户上传图 + 文本保留模板”的方式。

可以这样说：

```text
请为这个模板做一组稳定性测试设计。
必须包含三种测试方式：
1. 只用文字；
2. 只用用户上传图；
3. 用户上传图 + 原梗图。

每个测试都要说明：
1. 用了哪些参考图；
2. 没用哪些参考图；
3. 主要验证什么；
4. 需要观察哪些风险。
```

## 低清用户上传图怎么处理

用户上传图低清是正常情况，不要直接放弃。

只要还能看出主体，就可以继续用。此时业务判断重点从“完全长得一样”改成“保留主要识别特征”。

常见可保留特征：

- 宠物：品种、毛色、脸型、耳朵、体型、主要花纹、气质。
- 商品：颜色、形状、包装轮廓、明显标识、材质感。
- 人物：发型、服装、姿态、整体气质。不要要求像素级脸部一致，除非参考图足够清楚且流程允许。
- 物件：轮廓、颜色、材质、关键结构。

可以这样说：

```text
用户上传图比较低清，请先判断它是否还能作为主体参考。
如果可以，请告诉我能稳定保留哪些特征。
如果不可以，请告诉我是建议补图，还是只能做语义替换。
```

## 业务验收标准

看结果时，不要只问“好不好看”。建议按这几项判断：

1. 梗还成立吗
   - 第一眼是否能看懂主要画面。
   - 第二眼是否能感受到反差或笑点。
   - 原模板的荒诞感、松弛感、冲突感是否还在。

2. 主体是否对
   - 是否保留了用户上传主体的关键特征。
   - 是否变成了另一个物种、另一个商品或另一个角色。
   - 低清图场景下，是否至少保留了主要类别和气质。

3. 风格是否对
   - 原图是低清照片，就不要变成 3D、插画、广告大片。
   - 原图没有文字，就不要自动加文字。
   - 原图是小宠物轻微拟人，就不要变成完整人类角色。

4. 构图是否对
   - 主体位置、前后景关系、画面重心是否还像原模板。
   - 允许轻微变化，但不应变成完全新梗图。

5. 参考图使用是否说清楚
   - 是否用了用户上传图。
   - 是否用了原梗图。
   - 结果是在验证创意、身份一致性，还是模板复现能力。

## 常见错误

避免这样给 agent 下指令：

```text
帮我自由发挥生成一张类似的。
```

问题：太宽泛，agent 可能改变模板、场景、主体形态和笑点。

更好的说法：

```text
基于这个梗图模板生成 lv1-lv5。
保持原图的风格、构图关系、荒诞感和笑点公式。
不要加入原模板没有的外部剧情。
如果使用参考图，请明确说明使用了哪些参考图。
```

避免这样说：

```text
把这只宠物放进去，越像越好。
```

问题：没有说明模板边界，也没有要求说明参考图使用情况。

更好的说法：

```text
把用户上传的宠物图作为主体参考，套入这个梗图模板。
请保持宠物的主要身份特征，同时保持原梗图的风格和笑点。
生成前先说明：是否使用用户图、是否使用原梗图、各自的作用是什么。
```

## 推荐复制模板

### 模板分析

```text
使用 $meme-template-analyzer 分析这张梗图。
请用业务语言输出：
1. 核心笑点；
2. 必须保留的模板特征；
3. 可替换维度；
4. 不建议变化的内容；
5. 适合用户上传图替换的位置。
先不要生成图片。
```

### 用户图替换

```text
把用户上传图作为主体参考，套入这个梗图模板。
请先说明：
1. 用户上传图是否会作为主体参考；
2. 原梗图是否会作为生成参考图；
3. 如果原梗图不直接参与生成，会用哪些模板特征约束结果；
4. 用户图如果低清，能保留哪些主要特征；
5. 当前更适合验证创意方向还是最终质量。
```

### 创意等级对比

```text
基于这个梗图模板，输出 lv1-lv5 的变化方案。
要求：
1. 每个等级都保留原模板的风格、构图关系和笑点公式；
2. 每个等级说明会变化什么；
3. 每个等级说明不能变化什么；
4. 不要加入原模板没有的外部剧情；
5. 如果后续生成图片，必须说明参考图使用情况。
```

### 稳定性测试

```text
请为这个梗图模板设计稳定性测试。
必须包含：
1. 只用文字；
2. 只用用户上传图；
3. 用户上传图 + 原梗图。

每种方式都要说明：
1. 使用了哪些参考图；
2. 没有使用哪些参考图；
3. 主要验证什么；
4. 可能出现什么风险；
5. 业务上应该如何判断结果是否可用。
```

## 交付物怎么看

业务人员一般只需要关注这些内容：

- 这个模板适合什么内容。
- 哪些地方可以换。
- 哪些地方不能换。
- 用户上传图是否被用作主体参考。
- 原梗图是否被用作生成参考。
- `lv1-lv5` 的变化是否符合预期。
- 测试结果是否说明了风险。

不需要关注底层字段、文件结构或技术实现。如果输出里没有说清楚参考图使用情况，可以直接要求 agent 补充说明后再继续。
