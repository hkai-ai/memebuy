# 梗图模板库分类逻辑说明

## 1. 背景

当前“模板”主要指可以被复用、替换、扩展的梗图内容结构。新主流程不再要求业务先选择高保真或自由生成，而是先生成前端图片编辑模板，让用户通过槽位、候选项、上传图和整段提示词编辑完成输入。

## 2. 核心结论

当前建议采用：

> 模板是内容原型，场景是入口标签，系列是运营组织方式，子模板是稳定变体，前端编辑模板是用户交互层。

也就是说：

| 概念 | 作用 | 示例 |
| --- | --- | --- |
| 场景 | 用户搜索和进入的入口 | 宠物、情侣、儿童、亲子 |
| 系列 | 内容运营和专题组织方式 | 宠物变食物系列、情侣治愈系列 |
| 模板 | 具体梗图结构 / 内容原型 | 动物自然长成食物 |
| 子模板 / 变体 | 同一模板下的稳定变化 | 猫咪羊角包、企鹅饭团 |
| 前端编辑模板 | 用户实际操作界面 | 文本槽、图片槽、候选项、可重写提示词 |
| 历史生成模式 | 旧 prompt pipeline 的生成范围说明 | 高保真模式、自由生成模式 |

高保真/自由生成属于历史生成模式，不是新主流程的默认入口。

## 3. 场景是什么

场景是用户需求入口，例如：

- 宠物
- 情侣
- 儿童
- 亲子
- 闺蜜
- 职场
- 校园

用户搜索“宠物”，系统应该召回带有“宠物”标签的模板结果图或编辑模板。场景本质是标签系统，不是模板本身。

## 4. 系列是什么

系列是内容运营和专题组织方式，例如：

- 宠物变食物系列
- 情侣治愈系列
- 儿童搞笑系列
- 魔法动物系列
- 复古动物行为系列

一个系列里可以包含多个模板，一个模板也可以出现在多个系列里。

## 5. 模板是什么

模板是一个可以复用的梗图内容结构。它可以来自参考图，也可以来自纯文字 prompt 创意。

模板需要说明：

- 核心梗点。
- 观看逻辑。
- 第一眼 / 第二眼阅读模型。
- 可变维度。
- 不可变维度。
- 前端可编辑槽位。
- 后端拼接提示。

## 6. 前端编辑模板是什么

前端编辑模板是用户真正会操作的结构，主产物是 `image-edit-template.json`。

它包含：

- `templateText`: 带 `【槽位：默认值】` 的模板句。
- `editablePrompt`: 用户可整段编辑的提示词。
- `allowFullRewrite: true`: 允许用户整段删除、重写或只改槽位。
- `slots[]`: 文本槽、选择槽、长提示词槽、上传图槽、选择图槽。
- `suggestions`: 模型生成的候选替换项。
- `mockUserInput`: 示例用户输入。
- `backendHint`: 后端如何拼接指令。

前端编辑模板不要求用户理解高保真/自由生成。用户只需要知道能改什么、能传什么图、能不能整段改写。

## 7. 模板和场景的关系

一个模板可以属于多个场景，一个场景下也可以有多个模板。

例如“动物食物融合梗图”可以同时属于：

- 宠物
- 美食
- 儿童
- 治愈
- 表情包
- 小红书

前端可以按“宠物”展示，但底层不要把模板固定死在某一个场景下面。

## 8. 模板和系列的关系

一个系列里可以有多个模板。例如在“宠物变食物系列”里，可以有：

- 模板 A：动物自然长成食物。
- 模板 B：动物从食物里钻出来。
- 模板 C：动物在偷吃和自己长得很像的食物。
- 模板 D：动物伪装成商品包装。

这些可以属于同一个系列，但它们是不同模板，因为观看逻辑不同。

## 9. 如何判断两个内容是不是同一个模板

模板之间的区分，不是看画面表面有多像，也不是看属于哪个场景或系列，而是看以下核心是否变化：

- 核心梗点。
- 主体关系。
- 观看逻辑。
- 显著性逻辑。
- 画面结构。
- 用户理解到的笑点 / 反差点。

如果这些没有变，只是换主体、道具、食物、颜色或局部风格，那么它仍然属于同一个模板。前端可以把这些变化设计成槽位。

如果这些发生了变化，就应该成为一个新模板。

## 10. 子模板 / 变体是什么

子模板，也可以叫稳定变体。它仍然属于原模板，只是在某些维度上形成了比较稳定、可复用的组合。

例如：

```text
父模板：动物食物融合梗图

子模板 / 变体：
- 猫咪羊角包
- 企鹅饭团
- 兔子雪媚娘
- 青蛙抹茶布丁
```

这些可以作为 `suggestions`、示例作品或专题素材，但不一定要升级成一级模板。

## 11. 什么时候子模板应该升级为新模板

判断标准：

> 如果只是换主体、换食物、换颜色、换局部场景，它还是子模板。  
> 如果核心观看逻辑变了，它就是新模板。

仍然是子模板：

- 兔子雪媚娘
- 猫咪羊角包
- 企鹅饭团
- 狗狗泡芙

应该升级成新模板：

- 宠物在食物旁边装无辜。
- 动物从食物里钻出来。
- 食物变成动物的交通工具。
- 动物正在偷吃和自己长得一样的食物。

## 12. 历史生成模式

模板库不再区分高保真模式和自由生成模式；统一使用当前 GalleryTemplate 编辑与生成流程。

新主流程是：

```text
图片/创意
-> 识别核心公式
-> 生成 image-edit-template.json
-> 前端展示槽位和可编辑提示词
-> 用户改槽位、传图或整段重写
-> 后端拼接图像编辑指令
```

如果历史文档或后台字段里还出现 `generationFit`、`modes.hifi`、`modes.free`，它们只表示旧后台兼容和运营参考，不要求前端让用户选择模式。

## 13. 前端页面如何对应

### 瀑布流

瀑布流展示模板结果图或模板入口。每张图背后绑定：

- 所属模板。
- 所属子模板 / 变体。
- 场景标签。
- 主题标签。
- 风格标签。
- 情绪标签。
- 用途标签。
- 对应的 `image-edit-template.json`。

### 编辑页

编辑页读取 `image-edit-template.json`，展示：

- `editablePrompt` 完整编辑框。
- `slots[]` 表单。
- `suggestions` 候选项。
- 图片上传或选择控件。
- mock 预览。

用户可以只改槽位，也可以整段重写 prompt。

### 专题页

专题页是运营组织方式，不是底层分类方式。一个专题可以包含多个模板，一个模板也可以进入多个专题。

## 14. 一句话总结

> 场景是用户入口，系列是运营集合，模板是梗图结构，子模板是稳定变体，前端编辑模板是用户操作层；高保真 / 自由生成是历史生成模式，不是新主流程。

## 15. 标签层级与来源

标签不能只用一个字符串数组表达。当前管理台同时记录：

- 层级：`category` 代表运营大类，`tag` 代表内容或模板细标签。
- 来源：`operator` 代表运营选择，`template` 代表模板固定，`ai` 代表模型识别，`external` 代表 Pinterest、Instagram、Tumblr 等平台原始标签。

运营在管理台“标签词库”维护可选范围。AI 大类只能选择启用且 `aiAssignable` 的词库项；AI 自由生成只用于细标签。最终模板用 `metadata.tagAssignments` 保留来源，同时把已接受标签铺入 `metadata.tags` 兼容现有搜索。
