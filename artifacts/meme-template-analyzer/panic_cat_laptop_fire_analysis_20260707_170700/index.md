## VLM Recognition Mock

画面是一只黑色卡通猫在笔记本电脑前极度惊恐，背景和前景被夸张火焰包围。猫的眼睛巨大、瞳孔偏小，嘴巴张成橙红色大三角，耳朵竖起，胡须向两侧拉开。电脑位于右下前景，屏幕背面有爪印符号。火焰以高饱和红、黄、橙、紫笔触铺满背景，带有粗糙手绘油画/版画质感和强烈速度感。

## 阅读模型

- `first_read`：观众首先看到满屏火焰和猫的巨大惊恐表情，立即获得“事情失控”的情绪信号。
- `second_read`：视线落到右下角电脑，重新理解为“猫正在处理某个电脑/网络/工作任务，结果灾难化”。
- `reading_order`：火焰背景 -> 猫脸和眼睛 -> 大嘴 -> 电脑 -> 再回到火焰，形成循环式焦虑放大。
- `misdirection_or_reinterpretation`：火焰像真实灾难，也像心理状态或任务压力的视觉化。电脑让灾难从物理火场转为“屏幕里发生的事”。
- `salience_model`：最高显著性是火焰和猫脸；次级显著性是电脑；爪印符号只是装饰，不应成为笑点核心。

## 笑点机制

这张图的梗公式是“微小/日常触发器 + 过度灾难化反应”。电脑本身很普通，但猫的表情和火焰把它转译成生死级别危机。适合表达：

- 代码、部署、账单、邮件、考试、客服、运营事故等突然爆炸。
- 明明只是看了一眼屏幕，却像世界毁灭。
- 角色弱小、无助、可爱，但环境强烈失控，形成反差。

它不是靠文字包袱成立，而是靠图像情绪强度和“电脑作为事故入口”的角色映射成立。

## 可复用模板公式

`【主体：惊恐的小动物/角色】` 在 `【对象：电脑/屏幕/工作台设备】` 前看到 `【触发事件：任务、消息、报错、账单、数据、通知】`，背景用 `【情绪环境：火焰/爆炸/混乱能量】` 把内心崩溃外化。

## Prompt Style Profile

手绘夸张卡通插画，方形近景构图，粗黑线和干刷刮擦纹理，强烈的版画/油画棒质感。主体边缘毛躁，火焰用大面积斜向笔触和飞溅点构成。色彩以高亮黄色、橙色、红色为主，右侧混入洋红和紫色阴影。镜头贴近主体，几乎没有景深，视觉层级依靠大眼睛、黑色猫身和右下角电脑建立。负向漂移包括：写实猫、干净矢量图、柔和水彩、低对比配色、复杂背景细节、文字覆盖主体表情、过度精致的 3D 渲染。

## 适合的使用场景

- 程序员看到生产报错、账单飙升、CI 全红。
- 运营看到活动数据、广告花费或库存异常。
- 学生打开作业/成绩/考试范围。
- 创作者看到平台通知、评论区或截止日期。
- 普通用户看到银行账单、邮件收件箱或软件更新失败。

## 失效模式

- 表情不够夸张，梗会变成普通焦虑图。
- 火焰不包围主体，灾难感下降。
- 电脑或触发对象不清楚，观众无法理解“为什么崩溃”。
- 主体太强势或太冷静，会破坏“无助反应者”的角色。
- 添加太多文字会抢走图像驱动笑点。
- 换成真实灾难新闻语境时，幽默可能变得不合适。
